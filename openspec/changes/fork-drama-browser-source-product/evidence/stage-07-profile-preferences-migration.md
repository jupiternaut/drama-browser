# Stage 07 - Profile, Preferences, and Migration

## Scope

Stage 07 verifies that source-built Drama Browser owns profile and product preference identity before UI/runtime parity work continues.

## Profile Ownership

- Canonical source profile owner directory: `DramaBrowser`
- Canonical app profile name: `drama-browser`
- Canonical product display name: `Drama Browser`
- Expected fresh profile root on macOS: `~/Library/Application Support/DramaBrowser`
- Disallowed source-built product profile roots: `~/Library/Application Support/Zen`, `~/Library/Application Support/ZenDrama`, and `~/Library/Application Support/zen`

## Preference Ownership

- Canonical Drama Browser product preference namespace: `drama.browser.*`
- Legacy wrapper preference namespace: `zen.drama.*`
- `zen.drama.*` may be read only as a one-way migration/import source.
- Source-built Drama Browser must not write `zen.drama.*` as canonical product preferences.

## One-Way Migration Plan

On first source-built launch, `DramaBrowserChromeManager.mjs` imports any user-set `zen.drama.*` compatibility values into matching `drama.browser.*` preferences only when the canonical `drama.browser.*` value is absent. It then records `drama.browser.legacy-zen-pref-migrated=true`.

This is intentionally one-way:

- old wrapper profiles can keep their legacy keys;
- source-built Drama Browser writes only canonical `drama.browser.*` keys;
- repeated launches must not keep treating Zen as the product owner;
- future cleanup can remove legacy alias reads after the wrapper compatibility window closes.

## Verification

Machine-readable evidence is generated by:

```bash
bun run scripts/verify-drama-source-profile.ts \
  --source-workspace /Users/gengrf/zen-browser-desktop \
  --app-bundle "/Users/gengrf/zen-browser-desktop/engine/obj-aarch64-apple-darwin/dist/drama-browser/Drama Browser.app" \
  --out openspec/changes/fork-drama-browser-source-product/evidence/stage-07-source-profile.json
```

The verifier checks build config, `config.status`, built `application.ini`, source and packaged Chrome manager pref handling, one-way migration documentation, and a temporary explicit profile run under `DramaBrowser`.
